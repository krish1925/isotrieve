AECP Documentation

The Agent Embedding Communication Protocol enables agents with different embedding models to communicate semantic information directly, achieving 2x better semantic preservation compared to text serialization.

What is AECP?

When AI agents communicate today (like in MCP or other agent frameworks), they use text-to-text transfers. Approximately 57% of information is lost in this process. AECP reduces this to 14% information loss (86% preserved vs 43% with text).

Key Features

  • 97% fidelity on unseen vocabulary (vs 43% with text transfers)
  • 86% fidelity on unseen corpus sentences (vs 43% baseline)
  • <1ms transfer latency (vs 150ms+ with re-embedding)
  • Works with OpenAI, Voyage, Cohere, HuggingFace, and other embedding models
  • One-time calibration, then instant transfers

Quick Start

Get started with AECP in under 5 minutes. This guide walks you through creating your first agent communication.

Python

Here's a complete example showing how to set up two agents and transfer embeddings between them:

from aecp import AECP
from aecp.adapters import OpenAIAdapter, VoyageAdapter

# Step 1: Create two agents with different embedding models
agent1 = AECP(OpenAIAdapter(api_key="sk-your-openai-key"))
agent2 = AECP(VoyageAdapter(api_key="pa-your-voyage-key"))

# Step 2: Calibrate once (learns how to transfer between models)
print("Calibrating agents...")
calibration_result = agent1.calibrate_with(agent2)
print(f"Calibration quality: {calibration_result.validation_similarity:.2%}")

# Step 3: Transfer embeddings between agents
text = "machine learning and neural networks"
transfer = agent1.transfer_to(agent2.agent_id, text)

print(f"Original embedding dimensions: {agent1.embed(text).shape}")
print(f"Transferred embedding dimensions: {transfer.embedding.shape}")
print(f"Expected similarity: {transfer.expected_similarity:.2%}")

# Step 4: Use the transferred embedding in agent2's space
# The embedding is now compatible with agent2's model

Step-by-Step Explanation

  1. Create Agents: Initialize two AECP agents with different embedding providers.
  2. Calibrate: Run calibration once to learn transfer matrices between the two models.
  3. Transfer: Transfer embeddings from agent1's space to agent2's space.
  4. Use: The transferred embedding can now be used with agent2's native operations.

TypeScript

The same example in TypeScript/JavaScript:

import { AECP } from '@aecp/core';
import { OpenAIAdapter } from '@aecp/adapters-openai';
import { VoyageAdapter } from '@aecp/adapters-voyage';

async function quickStart() {
  // Step 1: Create two agents with different embedding models
  const agent1 = new AECP({
    embedder: new OpenAIAdapter({
      apiKey: process.env.OPENAI_API_KEY
    })
  });

  const agent2 = new AECP({
    embedder: new VoyageAdapter({
      apiKey: process.env.VOYAGE_API_KEY
    })
  });

  // Step 2: Calibrate once
  console.log("Calibrating agents...");
  const calibrationResult = await agent1.calibrateWith(agent2);
  console.log(`Calibration quality: ${calibrationResult.qualityMetrics.meanSimilarity.toFixed(2)}`);

  // Step 3: Transfer embeddings
  const text = "machine learning and neural networks";
  const transfer = await agent1.transferTo(agent2.agentId, text);

  console.log(`Transferred embedding dimensions: ${transfer.embedding.length}`);
  console.log(`Expected similarity: ${transfer.expectedSimilarity.toFixed(2)}`);
}

quickStart().catch(console.error);

Core Concepts

Understanding these concepts will help you use AECP effectively.

What is Calibration?

Calibration is a one-time process where two agents learn how to transfer embeddings between their different embedding spaces. Think of it as learning a translation dictionary between two languages.

The calibration process:

  1. Shared Vocabulary: Both agents encode the same set of vocabulary items
  2. Matrix Computation: AECP computes transfer matrices using least squares regression
  3. Quality Validation: The matrices are tested on held-out data to ensure quality

Calibration Configuration

Recommended settings:

  • Vocabulary size: 500–1000 for quick setup, 5000+ for production
  • Validation size: 10% of vocabulary size
  • Quality threshold: 0.75 minimum, 0.85+ ideal

Transfer Quality

Quality is measured using cosine similarity between the original and round-trip transferred embedding. Higher scores mean better semantic preservation.

Score Quality Description Action
> 0.90 Excellent Near-perfect semantic preservation Use confidently in production
0.80–0.90 Good High-quality transfer Production-ready
0.75–0.80 Acceptable Moderate quality Monitor closely, consider recalibration
< 0.75 Poor Low quality transfer Recalibration required

Information Preservation

AECP significantly improves information preservation compared to text-based transfers:

  • Text transfers: Only 43% of information survives the round trip
  • AECP vocabulary: 97% fidelity on 300k unseen vocabulary items
  • AECP corpus: 86% fidelity on 10k unseen sentences

When to Recalibrate

Recalibration is needed when:

  • Transfer matrix expires (default: 7 days)
  • Quality drops below threshold
  • Embedding models are updated
  • Domain shift occurs (e.g., switching from general to medical text)

Python Patterns

Cost Optimizer

from aecp.patterns import CostOptimizer
from aecp.adapters import OpenAIAdapter, VoyageAdapter

optimizer = CostOptimizer(
    cheap_adapter=OpenAIAdapter(model="text-embedding-3-small"),
    expensive_adapter=VoyageAdapter(model="voyage-large-2"),
)

optimizer.calibrate()

# Use cheap model by default
embedding = optimizer.embed("query", precision="low")

# Use expensive model when needed
embedding = optimizer.embed("critical query", precision="high")

# Check savings
stats = optimizer.get_stats()
print(f"Savings: {stats['savings_percentage']}%")

Privacy Bridge

from aecp.patterns import PrivacyBridge
from aecp.adapters import HuggingFaceAdapter, OpenAIAdapter

bridge = PrivacyBridge(
    local_adapter=HuggingFaceAdapter(),      # Runs locally
    cloud_adapter=OpenAIAdapter(api_key="sk-..."),
)

bridge.calibrate()

# Embed sensitive data locally (never leaves your server)
local_emb = bridge.embed_local("Patient SSN: 123-45-6789")

# Transfer only semantics to cloud
cloud_emb = bridge.transfer_to_cloud(local_emb)

Agent Handoff

from aecp.patterns import AgentHandoff
from aecp.adapters import VoyageAdapter, OpenAIAdapter, CohereAdapter

handoff = AgentHandoff({
    'code': VoyageAdapter(model='voyage-code-2'),
    'general': OpenAIAdapter(),
    'multilingual': CohereAdapter(model='embed-multilingual-v3.0'),
})

handoff.calibrate_all()

# Start task with code agent
context = handoff.start("Debug this Python code", agent='code')

# Hand off to general agent (context preserved)
context = handoff.transfer(context, to_agent='general')

Error Handling

try {
  const transfer = await agent1.transferTo(agent2.agentId, embedding);
} catch (error) {
  if (error.message.includes('No calibration found')) {
    await agent1.calibrateWith(agent2);
  } else if (error.message.includes('expired')) {
    await agent1.calibrateWith(agent2);
  } else {
    throw error;
  }
}

Plugin System

AECP includes a plugin system that allows you to create custom embedding adapters without modifying the core library.

Python

from aecp import register_adapter, adapter
from aecp.adapters.base import BaseAdapter

class MyCustomAdapter(BaseAdapter):
    def __init__(self, api_key: str, **kwargs):
        super().__init__(**kwargs)
        self.api_key = api_key
        self._dimensions = 384

    def _embed_impl(self, text: str) -> List[float]:
        # Your implementation here
        return [0.1] * self._dimensions

    def get_dimensions(self) -> int:
        return self._dimensions

    def get_model_id(self) -> str:
        return "my-provider:v1"

# Register it
register_adapter("my-provider", MyCustomAdapter)

# Or use decorator
@adapter("my-provider")
class MyCustomAdapter(BaseAdapter):
    # ... same implementation
    pass

TypeScript

import { registerAdapter, adapter } from '@aecp/core';
import { EmbeddingProvider } from '@aecp/core';

class MyAdapter implements EmbeddingProvider {
  async embed(text: string): Promise<number[]> {
    // Your implementation
    return [0.1, 0.2, ...];
  }

  async embedBatch(texts: string[]): Promise<number[][]> {
    return Promise.all(texts.map(t => this.embed(t)));
  }

  getDimensions(): number {
    return 384;
  }

  getModelId(): string {
    return 'my-model-v1';
  }
}

// Register it
registerAdapter("my-provider", MyAdapter);

// Or use decorator
@adapter("my-provider")
class MyAdapter implements EmbeddingProvider {
  // ... same implementation
}

Protocol Specification

Mathematical Foundation

Given two embedding spaces A and B with vocabularies V:

  1. Encode vocabulary: E_A = embed_A(V), E_B = embed_B(V)
  2. Solve least squares: W_AB = argmin ||E_A @ W - E_B||²
  3. Compute reverse: W_BA = argmin ||E_B @ W - E_A||²

Transfer Operation

To transfer embedding e_A from space A to space B:

e_B = e_A @ W_AB

Quality Guarantees

  • Training similarity: > 0.95 (round-trip on training vocabulary)
  • Validation similarity: > 0.80 (round-trip on held-out vocabulary)
  • Worst-case similarity: > 0.65 (minimum across validation)

Note: Both training and validation use round-trip evaluation (A→B→A) for fair comparison. Forward transfer (A→B only) is typically ~0.96 and is computed for reference but not used in quality comparisons.

Best Practices

1. Cache Calibrations

# Save transfer matrix
import json
matrix = agent1.transfer_matrices[agent2.agent_id]
with open('calibration.json', 'w') as f:
    json.dump(matrix, f)

# Load later
with open('calibration.json', 'r') as f:
    saved_matrix = json.load(f)

Troubleshooting

Low Quality Scores

Problem: Calibration quality < 0.75

Solutions:

  • Increase vocabulary size
  • Use more diverse vocabulary
  • Check if models are too different (e.g., 384D vs 3072D)
  • Consider using models from same family

Slow Calibration

Problem: Calibration takes too long

Solutions:

  • Reduce vocabulary size (minimum 200–300)
  • Use batch embedding APIs
  • Cache calibration results

Memory Issues

Problem: Out of memory during calibration

Solutions:

  • Reduce vocabulary size
  • Process in smaller batches
  • Use quantized models (HuggingFace)
