How AECP Works

Understanding the Agent Embedding Communication Protocol — from handshake to semantic transfer.

1

Handshake & Capability Exchange

Agents introduce themselves and exchange embedding model information:

  • Model name and version
  • Embedding dimensions
  • Quality thresholds
  • Protocol version
{
  "agent_id": "agent_a",
  "embedding_model": "text-embedding-3-small",
  "dimensions": 1536,
  "min_quality_threshold": 0.75
}
2

Calibration

Agents learn to translate between embedding spaces using shared vocabulary:

Training/Validation Split

  • 80% Training: Used to compute transfer matrices
  • 10% Validation: Held-out for quality checking
  • 10% Test: Completely unseen data

Transfer Matrix Computation

Uses least squares regression to find optimal linear transformation:

WAB = argmin ||EA · W - EB||²

Where EA and EB are embedding matrices from agents A and B.

3

Semantic Transfer

Once calibrated, embeddings can be transferred instantly:

<1ms latency 97% vocab / 86% corpus

Quality Metrics

  • Forward Similarity: How well A→B transfer preserves semantics
  • Round-trip Similarity: How well A→B→A preserves original
  • Worst-case: Minimum similarity observed
4

Monitoring & Recalibration

Continuous quality monitoring ensures reliable communication:

  • Automatic Expiration: Matrices expire after 7 days (configurable)
  • Quality Threshold: Alert if similarity drops below 75%
  • Transfer Logging: Track all transfers for debugging
  • Recalibration: Re-compute matrices when needed

Technical Deep Dive

Matrix Computation

AECP uses numpy's least squares solver (lstsq) which employs LAPACK's DGELSD:

  • Computes SVD for numerical stability
  • Handles over-determined systems (n_samples > dimensions)
  • Regularization available for ill-conditioned matrices
  • O(n · d²) complexity where n=samples, d=dimensions

Quality Guarantees

Validated on 300,000 vocabulary items with zero overfitting:

  • Training similarity (round-trip): 97.35% (on training vocabulary)
  • Validation similarity (round-trip): 97.34% (on held-out vocabulary)
  • Unseen vocabulary: 97.35% (completely unseen words)
  • Unseen corpus: 86.42% (complex sentences)
  • Worst-case: 82.43%

Key Finding: Training and validation both use round-trip evaluation for fair comparison. No degradation on unseen data (0.01% difference from validation).

Note: The gap between vocabulary (~97%) and corpus (~86%) reflects the natural complexity penalty when transferring complex multi-word sentences vs single vocabulary items.

Protocol Example 
# Handshake
agent1 = AECP(OpenAIAdapter())
agent2 = AECP(VoyageAdapter())

# Calibrate
result = agent1.calibrate_with(agent2)
print(f"Quality: {result.validation_similarity:.2%}")

# Transfer
transfer = agent1.transfer_to(
    agent2.agent_id,
    "machine learning"
)
// Handshake
const agent1 = new AECP({
  embedder: new OpenAIAdapter()
});
const agent2 = new AECP({
  embedder: new VoyageAdapter()
});

// Calibrate
const result = await agent1.calibrateWith(agent2);
console.log(`Quality: ${result.qualityMetrics.meanSimilarity}`);

// Transfer
const transfer = await agent1.transferTo(
  agent2.agentId,
  "machine learning"
);
